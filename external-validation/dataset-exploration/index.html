Panasonic, Samsung, and Graphite/LNMO Dataset Exploration

A presentation-style viability readout for three newly downloaded battery datasets. It explains what each dataset contains, what was plotted, and whether the data can support an unchanged external test of the CALB Section 33 rule.

Claim boundary: this page is not a validation result. It is a data-readiness screen. The frozen CALB Section 33 rule is still internal evidence until the same rule runs unchanged on an external dataset and beats a simple baseline model.

How To Read This Page

Section 33 is the frozen CALB pulse-to-EIS rule. In simple terms, it asks whether a time-domain current/voltage pulse can recover useful impedance-like information that agrees with EIS, which means Electrochemical Impedance Spectroscopy.

A dataset is useful for that test only if it has both sides of the bridge: same-cell EIS spectra and matching time-domain pulse or rest windows with current, voltage, temperature, and state labels. More files do not automatically mean better validation. A huge pulse dataset without EIS still cannot validate Section 33.

Frozen inputs means the rule is not allowed to change after seeing the external data. No tuning lambda, thresholds, windows, candidate rules, fallback logic, or pass criteria. That is the whole point. If we tune after looking, we are fooling ourselves.

Bottom Line

Panasonic is the only dataset here that currently deserves a real adapter attempt for frozen Section 33 inputs. It has EIS files and long current/voltage discharge windows that can plausibly be joined by cell state.

Samsung is useful for pulse-only exploration because it has large HPPC-style time-series archives. But no same-cell EIS was observed locally, so Samsung cannot validate Section 33 by itself.

Graphite/LNMO has clean EIS and useful electrochemical protocols, but it does not line up cleanly with the frozen Section 33 gate. The qOCV file gives zero accepted candidates, and the rate file gives only one long candidate.

None of the three is aging validation. Panasonic can test the pulse-to-EIS bridge on a different cell format and chemistry context. That is useful, but it is narrower than proving the CALB aging claim.

Panasonic 18650PF

The Panasonic archive has both EIS CSV files and MATLAB pulse files across five temperatures. That matters because Section 33 needs both an impedance spectrum and a time-domain current/voltage response.

The true five-pulse HPPC files are too short for the frozen Section 33 window rule because they are about 10 seconds long. The separate dis5_10p files are the important ones. Several of them contain long discharge windows with enough rest afterward to pass the frozen window gate.

Samsung INR21700-50E HPPC

The Samsung download is a large set of 7z archives with giant CSV reports. A streamed sample from Batch 1 shows usable current, voltage, cell temperature, and chamber temperature. That is good for pulse analysis.

The blocker is simple: no same-cell EIS was observed in the local files. Without EIS, there is nothing for Section 33 to compare the time-domain pulse result against. So Samsung is not a validation dataset for this rule unless a matching EIS source is found.

Graphite/LNMO

The graphite/LNMO archive is scientifically interesting, but weaker for this specific validation question. It has full-cell EIS by SOC, qOCV, rate, GITT-style files, and parameter JSONs. That is a rich electrochemistry package, but richness is not the same as Section 33 readiness.

The frozen window finder sees zero candidates in qOCV because the current is below the hard rest threshold. It sees one long candidate in the rate file, but that is not a clean HPPC-style EIS-pulse check-up join. So this dataset is useful for presentation and side experiments, not the first honest external Section 33 run.

Next Move

Build the Panasonic adapter first, using only the matched IDs 3541, 3623, and 3740. The adapter should create the exact inputs Section 33 expects: EIS spectrum, pulse/rest current and voltage window, temperature, state label, and candidate metadata.

After that, run the frozen rule unchanged and compare it against a simple baseline. Do not touch lambda, thresholds, windows, candidate logic, fallback logic, or pass criteria after seeing results. If the frozen run fails, call it failure or blocked evidence. Do not rescue it.
