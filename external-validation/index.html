Scope

This is dataset qualification, not validation.

The frozen CALB rule has not been externally validated here. These manifests only answer whether the local NASA and CALCE downloads have enough same-cell evidence to justify the next adapter.

Recommendation

Use CALCE Storage 25 C first.

Reason: CALCE has the cleaner external qualification path for the frozen CALB rule because it exposes PLN IDs, SOC labels, 25 C conditions, capacity workbooks, and same-cell impedance CSVs. The next adapter can make explicit same-PLN, same-SOC, same-temperature joins without playing games.

NASA should be second. It is not useless, but it is the wrong first fight. It has same-cell impedance plus charge/discharge cycling, not the pulse or drive-cycle structure that already broke confidence in Sections 36, 37, 40, and 42. If you start with NASA because it has more rows, you are optimizing for comfort instead of the actual failure mode. That would be bad science.

Claim Boundary

Allowed:

  • CALCE Storage 25 C is the better first external dataset for same-cell capacity plus EIS qualification.
  • NASA is usable later as a same-cell impedance plus charge/discharge robustness check.

Not allowed:

  • The CALB rule is externally validated.
  • NASA validates the drive-cycle bridge.
  • CALCE validates drive-cycle behavior. It does not.

Next Step

Build the CALCE adapter first:

1. Join pln_metadata.csv, capacity_workbook_channels.csv, and impedance_csv_manifest.csv.

2. Freeze the join keys before modeling: pln_id, temperature_c, storage_period, and soc_percent.

3. Start with the 24 candidate 25 C PLN IDs from summary.json.

4. Run the frozen CALB rule unchanged, then compare against a boring ECM baseline.

5. If DRT loses or the join is dirty, say that. Do not massage it into validation.
