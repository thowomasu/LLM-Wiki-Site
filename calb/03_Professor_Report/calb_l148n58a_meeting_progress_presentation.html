Meeting brief

CALB L148N58A Progress Update

Say this plainly: the project has strong internal CALB pulse-to-EIS bridge evidence. It does not yet have external validation. That is the honest line.

Use this framing: we built a working time-domain DRT pipeline, found a strong internal bridge result, tested the risky drive-cycle and external claims, and found where the evidence breaks. That is progress. Do not dress it up as full validation.

Current Progress

Section 33 quality-pass rows across 10, 25, and 40 C. Median bridge correlation is 0.928.

Drive-cycle transfer

External validation

CALCE is blocked. RADAR4KIT and Panasonic adapter runs currently return no support.

Questions To Ask In The Meeting

  1. Is internal same-cell CALB pulse-to-EIS bridge evidence enough for the current milestone, or do we need external validation before writing this up?
  2. Which external dataset should count as acceptable validation: CALCE, RADAR4KIT, or another dataset with same-cell EIS plus pulse/rest windows?
  3. What baseline must the method beat: voltage-only, ECM, or a published method?
  4. If drive-cycle transfer is still required, what SOC alignment rule should be treated as acceptable?
